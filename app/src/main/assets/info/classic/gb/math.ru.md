### Модель градиентного бустинга

Градиентный бустинг строит ансамбль слабых моделей (обычно деревьев) последовательно, каждая следующая модель исправляет ошибки предыдущих.

---

### Инициализация

Начальная модель $F_0(x)$ выбирается как константа, минимизирующая функцию потерь:

$$F_0(x) = \arg\min_\gamma \sum_{i=1}^m L(y_i, \gamma)$$

Для регрессии с квадратичной потерей это среднее значение $y$.

---

### Итерационный процесс

Для каждого шага $t = 1, \dots, T$:

1. **Вычислить остатки** (антиградиенты) для каждого объекта:

$$r_{it} = -\left[ \frac{\partial L(y_i, F_{t-1}(x_i))}{\partial F_{t-1}(x_i)} \right]$$

2. **Обучить слабую модель** $h_t(x)$ на данных $(x_i, r_{it})$ (например, дерево решений, предсказывающее остатки).

3. **Найти оптимальный шаг** $\rho_t$, минимизирующий потерю на новых предсказаниях:

$$\rho_t = \arg\min_\rho \sum_{i=1}^m L(y_i, F_{t-1}(x_i) + \rho h_t(x_i))$$

4. **Обновить модель**:

$$F_t(x) = F_{t-1}(x) + \nu \rho_t h_t(x)$$

где $\nu$ — скорость обучения (learning rate) в диапазоне (0,1], контролирующая вклад каждой новой модели.

---

### Регуляризация

Для предотвращения переобучения используются:
- Ограничение глубины деревьев.
- Минимальное число объектов в листе.
- Субдискретизация (использование части данных на каждом шаге).

Градиентный бустинг — один из самых мощных алгоритмов для табличных данных.
